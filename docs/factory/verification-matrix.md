# Verification matrix

This matrix is **generated** from `factory/inventories/verification.json`
and is mechanically checked against that file. It must not be
maintained by hand; the JSON is the source of truth.

The categories follow ACT-CLINEMM-FORK-BASELINE01 §I5:

| Class                | Meaning |
| -------------------- | ------- |
| `mandatory`          | Credential-free; must pass on every applicable authoritative host. |
| `affected-scope`     | Run only when the path filter in the source workflow matches. |
| `release-only`       | Publication or packaging. Not run in the baseline ACT. |
| `live-credentialed`  | Requires provider API credentials. Not run. |
| `manual-interactive` | Requires interactive UI / debugger. Not run. |
| `unsupported-on-host`| Cannot run on the current host class. |
| `obsolete`           | Replaced by another command. Retained for traceability. |
| `unknown`            | Reserved for unclassified commands. **Must be 0 at closure.** |

## Source-of-truth enforcement

`factory/scripts/verify-baseline.ts` is the canonical consistency
check. It enforces that:

- Every command's `source.path` resolves to an existing repository file.
- Every command's CSV / JSON row width matches the schema.
- The full list of `mandatory` and `affected-scope` commands is
  represented exactly once.

If this file and `verification.json` disagree, the JSON wins; the
markdown is regenerated by a successor ACT (see the matrix-regenerator
successor).
